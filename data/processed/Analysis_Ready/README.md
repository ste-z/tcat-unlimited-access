# Analysis-Ready Ridership

Generated by `scripts/build_analysis_ready_ridership.py`.

These are the canonical files for modeling the Cornell unlimited-access policy.
They are built directly from `data/raw/TCAT_Rides/` and
`data/raw/TCAT_GTFS/`; monthly cleaned ride caches are no longer required.

## Cleaning Rules

- Use `event_time = ts_event -> ts -> ts_vhist`.
- Require a resolved `normalized_route`.
  - The script resolves routes in memory using `Route_vhist` first, then GTFS
    validation of `corrected_route` only when `Route_vhist` is missing.
  - Through-running route strings, GTFS stop IDs in `corrected_route`, non-GTFS
    values, and missing route values are dropped rather than inferred.
- Keep revenue boarding records only.
- Drop `Empty TTP`.
- Drop duplicate-looking fare events using event time, bus, transaction
  sequence, fare text, fare description, and route.
- Treat `Stop_Id_vhist == 0` or blank as missing.
- Canonicalize stop names by stop ID.
- Keep blank APC-only rows for total ridership, but label them
  `Unspecified / No Description`.
- Treat blank-description `Got fare` records as ride-based regular fares.
- Keep `dropfile_parameter == EXCEPTION`, flag it, and do not treat it as
  invalid by itself.

## Files

- `ride_events_analysis_ready.parquet`: row-level retained ride events.
- `route_15min_panel.parquet`: route-by-15-minute panel.
- `route_stop_15min_panel.parquet`: route-stop-by-15-minute panel.
- `ride_events_analysis_ready.csv.gz`: CSV copy of row-level retained ride events.
- `route_15min_panel.csv`: CSV copy of the route-by-15-minute panel.
- `route_stop_15min_panel.csv`: CSV copy of the route-stop-by-15-minute panel.
- `route_group_lookup.csv`: route-level treatment/control labels.
- `route_stop_group_lookup.csv`: route-stop treatment/control labels.
- `stop_name_lookup.csv`: canonical stop-name lookup.
- `fare_category_lookup.csv`: fare-description recode lookup.
- `analysis_ready_cleaning_summary.csv`: row counts for retained and dropped
  analysis records.
- `analysis_record_issue_summary.csv`: issue flags before and after filtering.
- `route_resolution_summary.csv`: route-resolution status counts from raw ride
  files.
- `route_resolution_by_month.csv`: route-resolution status counts by source
  month.
- `route_column_coverage.csv`: coverage of raw route columns.
- `route_column_pairwise_disagreements.csv`: pairwise route-column agreement
  checks, restricted to values that are canonical GTFS route codes.
- `route_column_top_disagreements.csv`: largest raw route-column disagreements.
- `route_column_missing_patterns.csv`: missingness/canonical-route patterns
  across route columns.
