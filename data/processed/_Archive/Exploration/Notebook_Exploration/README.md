# Archived Exploration Outputs

These CSVs were generated during early notebook exploration of route codes,
fare-description/media consistency, and `dropfile_parameter` values. They are
kept for audit history, but the current notebooks and scripts should use:

- `data/processed/_Archive/Diagnostics/Ridership_Cleaning_Diagnostics/` for cleaning diagnostics.
- `data/processed/_Archive/Diagnostics/DiDisc_Group_Diagnostics/` for treatment/control diagnostics.
- `data/processed/Analysis_Ready/` for modeling inputs.

The files here are not deleted because some exploratory notebook cells may still
refer to them, but they should not be treated as canonical processed data.
