# Ridership Cleaning Diagnostics

Generated by `scripts/_archive_diagnostics/profile_ridership_cleaning_opportunities.py`.

This folder contains exploratory-but-repeatable checks that informed the final
analysis-ready cleaning rules. Use these files to explain why records are kept,
flagged, or dropped.

Important outputs:

- `quality_summary.csv`: high-level missingness and record-quality counts.
- `record_issue_summary.csv`: counts by issue flag.
- `record_issue_by_route.csv`: issue counts by normalized route.
- `event_time_source_summary.csv`: whether event time came from `ts_event`,
  `ts`, or `ts_vhist`.
- `duplicate_candidate_summary.csv` and `duplicate_candidate_rows_sample.csv`:
  duplicate-looking fare-event checks.
- `fare_category_lookup.csv`: fare recode source of truth copied from the
  taxonomy module.
- `fare_category_description_summary.csv`: observed descriptions within each
  category.
- `route_stop_didisc_group_candidates.csv`: route-stop exposure candidates before
  the final analysis-ready filtering.
